# Agent Notes

## Documentation Boundaries

- `docs/` is for user-facing documentation that can be published as the Vitify
  docs site.
- `contexts/` is for internal development context, planning notes, migration
  notes, and long-term design discussion.
- Do not publish internal planning context through the VitePress docs sidebar or
  navigation.
- When adding or moving project context, keep links in `README.md` and docs site
  navigation aligned with this split.
